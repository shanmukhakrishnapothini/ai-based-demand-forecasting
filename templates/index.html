About

AI Demand-Based Forecasting is a machine learning-powered web application designed to help businesses and analysts accurately predict future sales based on historical data. Using time series forecasting models like ARIMA, the project empowers users to make data-driven decisions in inventory planning, resource allocation, and market strategy.

This application offers a simple interface where users can upload sales data in CSV or Excel format and instantly visualize 30-day sales forecasts through interactive plots.

How it Works

How Your AI Demand Forecasting Project Works

User Uploads Data

The user visits your web page and is asked to upload a file — either a .csv or .xlsx (Excel) file. This file must contain two important columns

  • Date: the timeline of when sales occurred
  • Sales: the number of units or revenue generated on that date
Server Receives and Saves the File

Once the user uploads the file and hits submit:

  • Your Flask app checks the file type (must be CSV or Excel)
  • It saves the uploaded file into a folder on the server
  • This file is then read and converted into a structured time series using Pandas
Data Preprocessing

The uploaded data is cleaned and prepared:

  • The Date column is converted to a date format and set as the index
      • The time series is forced into a daily frequency, even if there are missing dates
  • This ensures the data is consistent for time-based forecasting
  • If any data points are missing due to frequency conversion, they may appear as gaps
Forecast Generation

After training the model on the user’s data:

  • The app asks the model to predict future sales for the next 30 days
  • These predicted values are stored and associated with future dates

This allows users to get an idea of what their sales might look like over the next month.

Plotting the Results

Now, to make the forecast easy to understand:

  • A line graph is generated
  • The left side of the graph shows actual historical sales (in blue)
  • The right side shows predicted sales for the next 30 days (in dashed red)

This visual makes it simple for users to:

  • See how well the model fits the existing data
  • Understand upcoming trends (growth, decline, or stability)
Displaying the Forecast on the Website

Once the plot is generated:

  • It’s saved as an image file on the server
  • That image is then displayed directly on the web page
  • Users can now see their forecast without any need for coding or technical knowledge
